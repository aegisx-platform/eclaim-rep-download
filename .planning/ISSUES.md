## Format

When adding issues, use this format:

```markdown

**Priority:** High/Medium/Low
**Phase:** [Which phase discovered]
**Category:** Bug/Enhancement/Technical Debt/Question

**Description:**
[What is the issue]

**Context:**
[Why it matters]

**Proposed Solution:**
[If known]

**Status:** Open/In Progress/Resolved/Deferred
```
