REP
REP - รายงานการเรียกเก็บ (Reimbursement Report)

รายงานที่ สปสช. ออกให้หน่วยบริการหลังประมวลผลข้อมูลการเบิกเรียบร้อยแล้ว

ข้อมูลสำคัญใน REP:
  • REP_NO: เลขที่งวดเบิก (เช่น 6801, 6802)
  • TRAN_ID: รหัสรายการธุรกรรมเฉพาะ
  • ข้อมูลผู้ป่วย: HN, AN, PID, ชื่อ-สกุล
  • ยอดเรียกเก็บ, ยอดชดเชย, DRG, AdjRW
ประเภทไฟล์ REP:
REP OP - ผู้ป่วยนอก (Outpatient)
REP IP - ผู้ป่วยใน (Inpatient)
REP ORF - ค่าใช้จ่ายส่งต่อ OP Refer
STM
Statement - ใบแจ้งยอด (Payment Statement)

เอกสารสรุปยอดเงินที่ สปสช. จะจ่ายให้หน่วยบริการในแต่ละงวด โดยรวมข้อมูลจาก REP หลายฉบับเข้าด้วยกัน

ข้อมูลใน Statement:
  • งวดการจ่าย (Statement Period)
  • รายละเอียดแยกตามกองทุนย่อย
  • ยอดหักต่างๆ (เงินเดือน, ค่า พ.ร.บ.)
  • ยอดสุทธิที่จะได้รับ
กระบวนการ:
  1. รวบรวมข้อมูลที่ผ่านการตรวจสอบ (สถานะ A)
  2. ตัดยอดข้อมูลตามกำหนดการ
  3. ประมวลผลคำนวณยอดชดเชย
  4. ออก Statement แจ้งหน่วยบริการ
SMT
SMT Budget - งบประมาณโอน (Smart Money Transfer)

ระบบบริหารการเบิกจ่ายเงินกองทุนหลักประกันสุขภาพแห่งชาติ ที่ สปสช. โอนเงินชดเชยให้หน่วยบริการ

ข้อมูลในระบบ SMT:
  • จำนวนงวดที่โอนในปีงบประมาณ
  • วันที่โอนเงิน
  • ยอดเงินที่โอน
  • รายละเอียดแยกตามประเภท OP/IP
ระบบที่เกี่ยวข้อง:
SMT Portal smt.nhso.go.th - ดูรายละเอียดการโอนแต่ละงวด
ความสัมพันธ์ระหว่าง REP_NO, TRAN_ID และข้อมูลต่างๆ 
TRAN_ID (รหัสรายการธุรกรรม)
   │
   │ ── รหัสเฉพาะของแต่ละ Visit/Episode
   │ ── ไม่ซ้ำกันในระบบ ใช้อ้างอิงตลอดกระบวนการ
   │
   └──→ REP_NO (เลขที่งวดเบิก)
           │
           │ ── รหัสงวดการเบิก (เช่น 6801, 6802)
           │ ── หนึ่ง REP_NO มีหลาย TRAN_ID
           │
           └──→ Statement Period (งวดใบแจ้งยอด)
                   │
                   │ ── รวม REP หลายฉบับ
                   │
                   └──→ SMT Transfer (การโอนเงิน)
                           │
                           └── โอนเงินตามยอด Statement

ตัวอย่าง: ผู้ป่วย HN: 12345 → TRAN_ID: 68012345001 → รวมใน REP_NO: 6801 → Statement งวด 1/68 → โอนเงินผ่าน SMT

ถูกปฏิเสธ

ยื่นอุทธรณ์หรือแก้ไขข้อมูล

สาเหตุที่พบบ่อยของการถูกปฏิเสธ
รหัสติด C (ข้อมูลไม่ถูกต้อง):
  • 101 ไม่มีชื่อ-สกุลผู้ป่วย
  • 104 เลขบัตรประชาชนไม่ถูกต้อง
  • 201 ไม่มีรหัสวินิจฉัยโรคหลัก
  • 301 ไม่มีค่าใช้จ่ายในการรักษา
รหัสติด Deny (ถูกปฏิเสธ):
  • 123 วันที่จำหน่ายหลังวันที่ได้รับข้อมูล
  • 156 รหัสสถานพยาบาลไม่มีในฐานข้อมูล
  • 159 รหัสโรงพยาบาลรับส่งต่อไม่ถูกต้อง
  • 182 รหัสหน่วยบริการไม่สอดคล้องกับสิทธิ

วิธีแก้ไข: ข้อมูลติด C แก้ไขผ่าน EPAC (ก่อนตัดยอด) หรือ e-Appeal (หลังตัดยอด)
